Interceptors

Types of Interceptors HttpClient supports

  1. functional
  2. DI based

Common Patterns of Interceptors

  1. Measuring server response times and log them.
    2. C0_LoggingInterceptor
  2. Adding authentication headers to outgoing requests to a particular API.
    3. C1_AuthAddInterceptor
  3. Retrying failed requests with exponential backoff.
    4. C2_RetryInterceptor
  4. Caching responses for a period of time, or until invalidated by mutations.
  5. Customizing the parsing of responses.
  6. Driving UI elements such as a loading spinner while network operations are in progress.
  7. Collecting and batch requests made within a certain timeframe.
  8. Automatically failing requests after a configurable deadline or timeout.
  9. Regularly polling the server and refreshing resolts.
